以下是用**最底层视角**整理的、结构清晰、语言尽量简洁的笔记，重点区分**内置Flash**与**无内置Flash（外置Flash）**两种典型单片机架构。

### 单片机最底层存储物理分类

| 介质          | 掉电保留 | 读速度   | 写/擦方式    | 典型容量     | 是否可XIP（直接执行代码） | 典型位置             |
| ------------- | -------- | -------- | ------------ | ------------ | ------------------------- | -------------------- |
| **内部Flash** | 是       | 中～快   | 扇区/页擦除  | 16KB～2MB    | 是（绝大多数支持）        | 芯片内部             |
| **内部SRAM**  | 否       | 极快     | 字节随机读写 | 2KB～512KB   | —                         | 芯片内部             |
| **外部Flash** | 是       | 慢～中   | 扇区/页擦除  | 1MB～256MB+  | 部分支持（需XiP/QSPI）    | 芯片外部（SPI/QSPI） |
| **外部RAM**   | 否       | 快～极快 | 字节随机读写 | 几MB～几十MB | —                         | 芯片外部（少见）     |

### 代码、变量在不同架构下的物理位置（运行时）

#### A. 主流架构：**内置Flash** 的单片片机（STM32、GD32、NXP、Renesas、多数AVR/PIC新款）

| 内容                      | 物理存储位置       | 链接器section | 运行时是否在RAM | 是否可直接执行 | 备注 / 启动时动作           |
| ------------------------- | ------------------ | ------------- | --------------- | -------------- | --------------------------- |
| 程序代码（函数、指令）    | **内部Flash**      | .text         | 否              | 是             | CPU直接从Flash取指          |
| const变量 / 字符串常量    | **内部Flash**      | .rodata       | 否              | —              | —                           |
| 全局变量（有初始值）      | **内部SRAM**       | .data         | 是              | —              | 启动代码：Flash → SRAM 拷贝 |
| 全局变量（无初始值 / =0） | **内部SRAM**       | .bss          | 是              | —              | 启动代码：SRAM对应区域清零  |
| static 变量               | **内部SRAM**       | .data 或 .bss | 是              | —              | 同全局变量                  |
| 局部变量、函数参数        | **内部SRAM（栈）** | 栈（动态）    | 是              | —              | SP向下生长，函数调用时压栈  |
| malloc分配的内存          | **内部SRAM（堆）** | 堆（动态）    | 是              | —              | 极少在小容量MCU使用         |
| 中断向量表                | **内部Flash**      | .isr_vector   | 否              | 是             | 通常放在Flash最低地址       |

**一句话总结**：代码和常量在**Flash**里直接运行，**所有可变数据**都在**内部RAM**。

#### B. 无内置Flash（或极小Flash）的单片机（部分低端、老款、某些超低功耗或大容量需求场景）

典型代表：部分早期8051、某些低端PIC、一些带外部QSPI Flash的Cortex-M（如部分SAMD51、i.MX RT部分型号、某些ESP32-S3的扩展模式等）

| 内容                  | 物理存储位置            | 是否可XIP     | 运行时主要位置                         | 启动/运行特点                            | 优缺点对比内置Flash架构                    |
| --------------------- | ----------------------- | ------------- | -------------------------------------- | ---------------------------------------- | ------------------------------------------ |
| 程序代码（大部分）    | **外部Flash**           | 支持XiP时：是 | 外部Flash（直接取指）或先拷贝到内部RAM | 需初始化QSPI/OSPI外设 → 映射到地址空间   | 容量大，但启动慢、功耗高、成本高、安全性低 |
| 少量关键代码/引导程序 | **极小内部Flash / ROM** | 是            | 内部Flash/ROM                          | 先运行bootloader，再跳外部               | —                                          |
| const / 常量表        | **外部Flash**           | —             | 外部Flash                              | 通过XiP直接读或拷贝到RAM                 | —                                          |
| 全局/静态变量         | **内部SRAM**            | —             | 内部SRAM                               | 启动时从外部Flash拷贝.data段（或不拷贝） | 拷贝时间更长，.data容量受限                |
| 局部变量、栈          | **内部SRAM**            | —             | 内部SRAM                               | 同内置Flash架构                          | —                                          |
| 大数组 / 缓冲区       | 内部SRAM 或 外部PSRAM   | —             | 视情况                                 | 很多这种架构会外接PSRAM来扩展可变数据    | 内部RAM通常更小                            |

**一句话总结**：代码主要在**外部Flash**（支持XiP时直接执行），变量仍主要在**内部RAM**，启动复杂、延迟更高。

### 内置Flash vs 无内置Flash（外置Flash）核心区别对比表

| 维度                | 内置Flash MCU                            | 外置Flash + 小/无内置Flash MCU                  | 典型应用场景                    |
| ------------------- | ---------------------------------------- | ----------------------------------------------- | ------------------------------- |
| 程序存储容量        | 受限（几十KB～2MB）                      | 可很大（8MB～256MB+）                           | 大型固件、OTA、AI模型、文件系统 |
| 直接执行代码（XIP） | 几乎全部支持                             | 部分支持（需QSPI/OSPI + XiP模式）               | —                               |
| 启动速度            | 极快（几十μs～几ms）                     | 较慢（需初始化外部接口，几十ms～数百ms）        | 快速上电 vs 大容量需求          |
| 功耗（运行时）      | 较低（内部总线）                         | 较高（外部IO翻转 + Flash读取电流）              | 低功耗产品 vs 高性能            |
| 安全性              | 较高（难物理读取）                       | 较低（外部Flash易被拆下读取/改写）              | 安全性要求高 vs 成本敏感        |
| 成本（芯片+外围）   | 通常较高（集成度高）                     | 芯片便宜 + Flash便宜，但PCB面积/层数增加        | —                               |
| 擦写寿命影响        | 程序区基本不擦，数据区慎用               | 数据区常擦写更方便（外部Flash寿命独立）         | 少量配置 vs 频繁写日志/参数     |
| 典型代表            | STM32F1/F4、GD32、CH32、ATmega、PIC16/18 | i.MX RT部分、SAMD51 + QSPI Flash、ESP32部分模式 | —                               |

### 极简记忆口诀

- **内置Flash主流机**：代码住Flash里跑，变量全塞内部RAM
- **外置Flash机**：代码大多住外面（可直接跑或先搬家），变量还是靠内部那点可怜的RAM
