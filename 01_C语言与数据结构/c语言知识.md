# C 语言模块化开发核心规范笔记

## 一、变量定义与赋值（模块化场景）

### 1. 核心原则

- 最小作用域：变量仅在需要范围生效，减少冲突；
- 显式初始化：局部变量必初始化，全局 / 静态变量推荐显式写 0；
- 模块化分工：.c 定义，.h 仅声明（跨模块）或不声明（文件私有）。

### 2. 规范写法

|     变量类型     |    定义位置     |    修饰符    |     对外暴露      |
| :--------------: | :-------------: | :----------: | :---------------: |
|     局部变量     | 函数 / 代码块内 | auto（默认） |  仅作用域内可见   |
| 文件私有全局变量 |     .c 顶部     |    static    |     无需暴露      |
|  跨模块全局变量  |   唯一.c 文件   |      无      | .h 用`extern`声明 |

## 二、全局变量

### 1. 定义

所有函数外部定义的变量，存储于全局 / 静态区，全程有效。

### 2. 核心特性

- 初始化：编译器自动赋 0/NULL（推荐显式写 0）；
- 作用域：`static`→文件私有（推荐），无`static`→跨模块可见（慎用）。

### 3. 禁忌

- 禁止.h 定义全局变量；
- 跨模块变量优先通过 get/set 函数操作。

## 三、模块化：运行时赋值的数组

### 1. 核心规则

- 定义：.c 中（私有加`static`，跨模块不加）；
- 赋值：封装`init`函数运行时赋值（如从 Flash 读取）；
- 访问：提供 get/set 接口，加越界 / 空指针检查。

### 2. 极简示例

c



运行









```
// sensor_param.h
#define CALIB_LEN 10
int sensor_calib_array_init(void);
int sensor_calib_array_get(int idx, float *val);

// sensor_param.c
static float g_calib_arr[CALIB_LEN] = {0};
int sensor_calib_array_init(void) {
    memcpy(g_calib_arr, read_flash_data(), sizeof(g_calib_arr));
    return 0;
}
```

## 四、#define 宏（如 FLASH_NTC_ADDR）

### 1. 内存特性

- 本质：预处理文本替换，无独立内存地址，不占 RAM/Flash；
- 数值去向：替换后融入使用场景（指令段 / 栈区 / 全局区）。

### 2. 适用场景

|         场景          | 是否用 #define |    替代方案    |
| :-------------------: | :------------: | :------------: |
|  魔法数字 / 硬件地址  |       ✅        |       -        |
|     条件编译开关      |       ✅        |       -        |
|      简单宏函数       |       ✅        |       -        |
| 类型安全 / 需取址常量 |       ❌        | `static const` |
|       关联常量        |       ❌        |   枚举`enum`   |

### 3. 存放位置

- 跨模块共享：.h 文件（加头文件保护，命名加模块前缀）；
- 文件私有：.c 文件（避免全局污染）；
- 全局编译开关：集中放`config.h`。

## 核心总结

1. 模块化：定义藏.c，声明 / 接口放.h，私有加 static；
2. 变量 / 数组：显式初始化、最小作用域、接口化访问；
3. \#define：编译期替换，共享放.h，私有放.c，避类型 / 复杂逻辑场景。
